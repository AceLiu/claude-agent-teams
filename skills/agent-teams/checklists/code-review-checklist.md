> Agent Teams v4.0.0 | 共 248 项 | 技术栈：Vue 3 + TypeScript（前端）+ Java/Spring/Dubbo（后端）
> Phase 5.5 代码质量深度审查时，根据变更涉及的模块**动态筛选**对应检查项。
> 未涉及的模块跳过，避免噪音。

## 使用方式

1. Phase 5.0 变更识别后，确定变更涉及哪些模块
2. 按下方路由表筛选需要检查的章节
3. Architect/Reviewer 逐条检查，标记 PASS / FAIL / N/A
4. FAIL 项按 P0/P1/P2 分级，写入 `quality-deep.md`

| 231 | @Async 异常处理 | P1 | `@Async` 方法抛出的异常默认被吞掉。是否配置了 `AsyncUncaughtExceptionHandler`？ |
| 232 | 延迟任务实现 | P1 | 延迟任务是否用了 `Thread.sleep`？应使用 ScheduledExecutorService / MQ 延迟消息 / Redis 过期事件 |
| 233 | 事务与消息一致 | P0 | 数据库写入和 MQ 发送是否在同一事务中？事务提交前发消息？应使用本地消息表或事务消息 |
| 234 | 消费者启动顺序 | P2 | 应用启动时消费者是否会消费到初始化未完成时的消息？是否需要延迟注册消费者？ |
| 235 | 消息序列化 | P1 | 消息体的序列化方式（JSON/Protobuf/Hessian）是否与消费方一致？版本升级时是否向后兼容？ |
| 236 | 重试风暴 | P1 | 消费失败重试是否有退避策略（exponential backoff）？固定间隔重试在下游故障时会造成重试风暴 |
